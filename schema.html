Tables & columns

Six tables in two schemas. Every column explained. The L1 primitives hold the facts; the L2 audit stream holds the history of how those facts came to be.

L1 · Primitives · public schema
public.
agents
Everyone and everything that participates in the cooperative's economic life. A row exists for each member (natural person), for each partner organization (legal entity), and for each AI agent (Nou, Dianoia). The cooperative itself — RegenHub, LCA — is also an agent in this table. This makes the database self-referential in a meaningful way: the entity that owns the database is a row within it.
Defined
ColumnTypeNotes
id PKUUIDGenerated. Stable. Referenced by role_assignments, resources, audit.events.
kindTEXTnatural | legal | agent_ai. Determines which RLS policies apply.
display_nameTEXTRequired. Must be non-empty after trimming.
emailCITEXTCase-insensitive. Unique. Null for AI agents.
auth_user_id FK → auth.usersUUIDNull until Phase 3 (Supabase Auth wiring). SET NULL on auth deletion — agent record persists.
metadataJSONBExtension field. AI agents store ENS, chain, ERC-8004 ID here.
created_atTIMESTAMPTZSet at insert. Never modified.
updated_atTIMESTAMPTZMaintained by trigger. Changes when display_name or metadata updates.
archived_atTIMESTAMPTZSoft delete. NULL = active. An agent.archived event sets this.
Lifecycle: agent.created → agent.email_verified → agent.archived
public.
agreements
The document registry. Every binding commitment — bylaws, member agreements, hub membership agreements, the EE grant letter, minutes, motions — has a row here. The actual clause text lives in agreement_sections (the child table). This table is the envelope; sections are the content. Amendments are new rows with a superseded_by link pointing back to the version being replaced.
Defined
ColumnTypeNotes
id PKUUIDGenerated.
kindTEXTbylaws | member_agreement | hub_member_agreement | grant | minutes | motion | employment. Open enum.
titleTEXTHuman-readable document name.
versionTEXTe.g. "v2", "v2.3", "2026-05-22". Displayed alongside title.
effective_atTIMESTAMPTZWhen this agreement came into force. Distinct from created_at.
superseded_by FK → agreementsUUIDSelf-referential. NULL = current version. Follow the chain to reach the latest.

metadataJSONBFlexible. Used for grant metadata, signatory lists, etc.
created_atTIMESTAMPTZSet at insert.
archived_atTIMESTAMPTZSoft archive. Active agreements have NULL.
Lifecycle: agreement.published → agreement.superseded
public.
agreement_sections
The bylaws as queryable rows. Every article, every section, every clause is a row in this table. The address column gives each section a stable identifier — bylaws:v2:art-1.sec-4 — that survives renumbering and amendments. When a section changes in a new bylaw version, the old row is not deleted; a new row is created and the old one gets a superseded_by link. This means Nou can cite bylaws:v2:art-18.sec-1 and the citation remains valid even after bylaws v3 is adopted.
Defined
ColumnTypeNotes
id PKUUIDGenerated.
agreement_id FK → agreementsUUIDRequired. ON DELETE RESTRICT — can't delete a document while its sections exist.
addressTEXTStable identifier: kind:version:segment. e.g. "bylaws:v2:art-1.sec-4". Unique per agreement.
ordinalINTEGERPosition for ordered traversal. Reconstruct the document by ORDER BY ordinal.
headingTEXTSection heading. Nullable (preamble, articles without sub-sections).
body_markdownTEXTFull section text in Markdown.
body_textTEXT GENERATEDPlain text for FTS. Generated from body_markdown. Used by to_tsvector() index.
embeddingVECTOR(1536)Semantic embedding for Nou's retrieval pipeline. Populated async in Wave 0 Phase 4. NULL until then.
superseded_by FK → agreement_sectionsUUIDPoints to replacement in a later version. NULL = current clause.
created_atTIMESTAMPTZSet at insert. Immutable.
Indexes: address lookup · ordinal traversal · FTS (GIN on body_text) · current-only partial index
public.
resources
Economic things with value that agents hold or control. A member's capital account balance, their equity share, a co-working seat, a treasury holding, accumulated CLOUD credits — all are rows in this table. The kind column distinguishes them. The quantity column is a fast-read snapshot; the authoritative value is always derived from the event stream. Think of this as the balance sheet, not the ledger.
Defined
ColumnTypeNotes
id PKUUIDGenerated.
kindTEXTcapital_balance | equity_share | hub_seat | treasury_holding | cloud_credit. Open enum.
owner_agent_id FK → agentsUUIDWho holds or controls this resource. May be RegenHub itself (cooperative-owned resources).
governing_agreement_id FK → agreementsUUIDAgreement that governs this resource. e.g. Patron Member Agreement for a capital_balance.
quantityNUMERIC(20,8)Snapshot value. Precise to 8 decimal places for CLOUD credits and fractions.
unitTEXT'USD', 'CLOUD', 'seat', 'share', etc.
metadataJSONBKind-specific extension data.
created_at / updated_at / archived_atTIMESTAMPTZStandard lifecycle columns. updated_at maintained by trigger.
Note: quantity is a snapshot. Authoritative value = sum of capital events in audit.events.
public.
role_assignments
The source of truth for who has what role, and when. Every role grant and revocation is a row here. Revocation sets the revoked_at column; no row is ever deleted. This means you can reconstruct the exact role an agent held at any moment in history: granted_at ≤ moment AND (revoked_at IS NULL OR revoked_at > moment). Every RLS policy in the database joins to this table to determine what the invoking user is permitted to do.
Defined
ColumnTypeNotes
id PKUUIDGenerated.
agent_id FK → agentsUUIDRequired. The agent receiving the role.
auth_user_id FK → auth.usersUUIDRedundant with agents.auth_user_id for fast RLS lookup without an extra join. Kept in sync by trigger.
roleTEXTOne of 15 roles. See Roles page for full list and descriptions.
granted_atTIMESTAMPTZWhen the role was granted. Defaults to now().
revoked_atTIMESTAMPTZNULL = active grant. Constraint: revoked_at must be after granted_at.
granted_by_agent_id FK → agentsUUIDWho authorized the grant.
governing_agreement_id FK → agreementsUUIDThe agreement that authorizes this role. e.g. bylaws for director role.
metadataJSONBExtension data. Bootstrap note in the Nou row.
created_atTIMESTAMPTZSet at insert.
Key index: Partial index on (auth_user_id, role) WHERE revoked_at IS NULL — used by every RLS policy evaluation
L2 · Audit · audit schema
audit.
events
The cooperative's permanent record. Every state change, every access decision, every Nou query is a row here. The table is append-only — a database trigger prevents any UPDATE or DELETE at the engine level, not just the policy level. No role can bypass it. This is simultaneously the REA economic event log (capital.contributed, agreement.signed) and the operational audit trail (role.assigned, access.denied) — one stream, readable as a continuous business narrative without joins.
Defined
ColumnTypeNotes
id PKUUIDGenerated. Immutable once written.
occurred_atTIMESTAMPTZWhen the event happened. Indexed DESC for recency queries.
actor_id FK → agentsUUIDWho caused this event. NULL for system events (migrations, cron jobs).
actor_roleTEXTRole held at the moment of the event. Stored directly — role_assignments may change later.
event_typeTEXTDot-namespaced past-tense verb. e.g. agreement.signed, capital.contributed, access.denied.
target_tableTEXTSchema-qualified table name. e.g. public.agents, audit.events.
target_idUUIDThe row that changed or was the subject of the event.
before_stateJSONBRow snapshot before the event. NULL for INSERT events.
after_stateJSONBRow snapshot after the event. NULL for DELETE events.
policy_citationTEXTStable bylaw address authorizing this event. e.g. "bylaws:v2:art-18.sec-1". Populated from Phase 2.
attestation_uriTEXTEAS on-chain attestation URI. NULL until Wave 3 attestation begins.
payloadJSONBEvent-type-specific data. Phase, milestone notes, custom fields per event namespace.
Enforced append-only: BEFORE UPDATE and BEFORE DELETE triggers raise an exception. No role or policy can bypass this.
